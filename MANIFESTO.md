# Ananya

> An Engineering Resource Platform

---

# Why Ananya Exists

Engineering teams spend too much time searching.

Searching for components.

Searching for tools.

Searching for documentation.

Searching for previous projects.

Searching for decisions someone made six months ago.

Searching for the same mistake that has already been solved.

Inventory is only one part of the problem.

Knowledge is the other.

Ananya exists to remember everything about an engineering workspace so that engineers can focus on building instead of searching.

---

# Our Mission

Help engineers build better things with less friction.

If a feature does not help someone build something faster, safer, or with fewer mistakes, it probably does not belong in Ananya.

---

# Our Vision

Ananya is not an ERP.

Ananya is not an inventory application.

Ananya is an Engineering Resource Platform.

It is the operating system for engineering teams.

---

# Our Philosophy

Everything important is an Object.

Objects gain behaviour through Capabilities.

Objects are connected through Relationships.

Knowledge is as valuable as Inventory.

History is as valuable as Current State.

---

# What We Believe

Inventory without knowledge is incomplete.

Knowledge without context is forgotten.

Relationships create understanding.

Good software should remember so humans don't have to.

---

# The Four Pillars

## 1. Objects

Everything meaningful is represented as an Object.

Examples

- Component
- Tool
- Machine
- Project
- Assembly
- Supplier
- Manufacturer
- Person
- Document
- Location

Objects are first-class citizens.

---

## 2. Capabilities

Objects are not defined by inheritance.

Objects are composed from capabilities.

Examples

- Stock
- Purchasing
- Manufacturing
- Maintenance
- Reservations
- Knowledge
- Attachments
- Relationships
- Automation
- Versioning

Capabilities define behaviour.

---

## 3. Relationships

Relationships are first-class.

Examples

ESP32

- stored in Drawer D02
- used in Weather Station
- compatible with SHT31
- purchased from LCSC
- manufactured by Espressif
- alternative is ESP32-S3

Nothing exists in isolation.

---

## 4. Knowledge

Every object accumulates knowledge.

Datasheets.

Notes.

Lessons learned.

Known issues.

Alternatives.

Engineering decisions.

Knowledge grows over time.

# UX Principles

Speed over complexity.

Search over navigation.

Visual over textual.

Relationships over hierarchy.

Knowledge over documentation.

Every important action should take less than three seconds.

---

# The Three Second Rule

An engineer should be able to locate any resource within three seconds.

Resource means more than stock.

It includes

- Components
- Projects
- Documents
- Machines
- Drawers
- Suppliers
- Knowledge

---

# Build, Don't Count

Traditional inventory systems optimise for counting.

Ananya optimises for building.

The question is never

"How many do we have?"

The question is

"Can we build it?"

# AI

Artificial Intelligence should never replace engineering judgement.

Its purpose is to reduce repetitive work.

AI should help discover relationships, organise knowledge, answer questions, and surface forgotten information.

The engineer always makes the decision.
