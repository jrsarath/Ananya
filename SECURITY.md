# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within Ananya, please follow these steps:

1. **Do not create a public GitHub issue** for security vulnerabilities
2. Send an email to [oss@48studios.in](mailto:oss@48studios.in) with the subject "SECURITY: [Brief Description]"
3. Include as much information as possible about the vulnerability:
   - Type of vulnerability
   - Steps to reproduce
   - Potential impact
   - Any related links or references

## Response Process

- You will receive an initial acknowledgment within 24 hours
- We aim to provide a detailed response within 5 business days
- We will keep you informed about our progress in fixing the issue

## Security Updates

Security updates will be released as patches and will be documented in the changelog.

## Supported Versions

We currently support security updates for:

- The latest stable release
- The previous major version (if still actively maintained)

## Disclosure Policy

We follow responsible disclosure practices. We will not publicly disclose vulnerabilities until a fix has been released and users have had time to update.
